# Documentation Migration Plan

## Current Status: Hybrid Approach

All planning documents use summary tables with details in the same file.

### When to Migrate to Separate Files

**Backlog (`backlog.md`):**
- Migrate when: **> 20 items**
- Create: `backlog/` directory
- Structure: One file per feature
- Table stays in `backlog.md` with links to individual files

**Index (`index.md`):**
- Migrate when: **> 15 completed features**
- Create: `completed/` directory
- Structure: One file per completed feature
- Table stays in `index.md` with links to individual files

### Migration Steps

When threshold is reached:

1. **Create directory:**
   ```bash
   mkdir -p docs/planning/backlog/
   # or
   mkdir -p docs/planning/completed/
   ```

2. **Move details to individual files:**
   - Keep table in main file
   - Update links to point to separate files
   - Use kebab-case for filenames (e.g., `rate-limiting.md`)

3. **Update links:**
   - Change ](#section)` to `[Details](./backlog/feature-name.md)`from `[

4. **Template for individual files:**
   ```markdown
   # Feature Name
   
   **Status:** Planned/Complete
   **Priority:** High/Medium/Low
   **Effort:** Small/Medium/Large
   
   [Full content here]
   
   **Related:**
   - [Backlog](../backlog.md)
   - [Other related items]
   ```
